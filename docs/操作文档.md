# Anime Subtitle Manager 操作文档

本文档用于说明当前版本已经实现的功能、运行方式和主要操作流程。

## 1. 当前实现范围

当前应用已经搭建为 Windows 桌面工具：

- 前端：Tauri v2 + React + TypeScript + Vite
- 后端：Rust
- 视觉：使用 `assets/` 中的图标、背景线稿、Logo、侧边角色图
- 目标：本地动漫视频/字幕扫描、匹配、整理和 MPV 启动

已实现的主要功能：

- 选择多个视频目录
- 选择多个字幕目录
- 选择一个输出目录
- 扫描 `.mkv`、`.mp4` 视频文件
- 扫描 `.ass`、`.ssa`、`.srt`、`.vtt` 字幕文件
- 从文件名和路径中解析季/集信息
- 从文件名和父目录中检测字幕语言
- 按集数匹配视频和字幕
- 在表格中展示匹配结果
- 为每集选择主字幕和副字幕
- MPV 启动时最多传入两个字幕文件
- 生成整理计划
- 支持复制整理和移动整理
- 目标文件冲突时不静默覆盖
- 冲突项可选择跳过、替换或重命名
- 生成 `anime-sub-map.json`
- 保存项目配置到输出目录
- 使用 Windows Explorer 打开视频或字幕位置

## 2. 快速启动

首次使用前安装依赖：

```powershell
corepack enable
corepack prepare pnpm@latest --activate
pnpm install
```

如果 pnpm 提示依赖构建脚本被拦截，可以执行：

```powershell
pnpm approve-builds --all
pnpm install
```

启动桌面应用：

```powershell
pnpm tauri dev
```

也可以运行已经构建过的调试版：

```powershell
src-tauri\target\debug\mpv-tidy.exe
```

浏览器预览只适合查看界面：

```powershell
pnpm dev
```

注意：浏览器预览没有 Tauri 本地能力，不能真正打开系统目录选择、扫描文件、整理文件或启动 MPV。

## 3. 主界面说明

界面参考设计图实现为三栏桌面工作台：

- 左侧：应用标识、导航菜单、侧边角色图、平台说明
- 中间：项目标题、目录选择、统计卡片、匹配表格、整理操作、输出结构预览
- 右侧：当前集详情、字幕候选、播放设置、操作按钮

顶部按钮：

- `保存项目`：将项目配置保存到输出目录
- `开始整理`：默认生成复制整理计划
- `用 MPV 播放`：播放当前选中的集数

底部操作：

- `复制整理`
- `移动整理`
- `生成映射文件`
- `刷新扫描结果`

## 4. 推荐操作流程

1. 在顶部项目名输入框确认项目名和季，例如 `Jujutsu Kaisen S01`。
2. 点击 `视频目录` 卡片中的 `添加目录`，选择一个或多个视频目录。
3. 点击 `字幕目录` 卡片中的 `选择目录`，选择一个或多个字幕目录。
4. 点击 `输出目录` 卡片中的 `选择目录`，选择整理后的目标目录。
5. 点击 `刷新扫描结果`。
6. 在匹配表格中查看每集状态。
7. 点击表格中的某一集，在右侧详情面板确认主字幕和副字幕。
8. 点击 `复制整理` 或 `移动整理` 生成整理计划。
9. 如果出现冲突，在整理确认弹窗中选择：
   - `跳过`
   - `替换`
   - `重命名`
10. 点击 `确认执行`。
11. 整理完成后检查输出目录和 `anime-sub-map.json`。
12. 选择某一集，点击 `立即播放` 或 `用 MPV 播放`。

## 5. 扫描与解析规则

视频扩展名：

```text
.mkv
.mp4
```

字幕扩展名：

```text
.ass
.ssa
.srt
.vtt
```

当前支持的集数识别示例：

```text
S01E01
S1E1
第01话
第1话
第01話
第1話
[01]
- 01
- 01v2
EP01
E01
```

解析器会尽量避免把以下内容误判为集数：

```text
1080p
720p
2160p
264
265
10bit
8bit
2020
2025
[2D6390A9]
H.264
```

如果没有识别到季数，默认按 `S01` 处理。

## 6. 字幕语言检测

当前会结合文件名和父目录名检测语言。

支持的主要语言：

```text
zh-Hans
zh-Hant
ja
en
und
```

示例映射：

- `简中`、`简体`、`SC`、`CHS`、`zh-Hans`、`zh-CN`、`zh_cn` → `zh-Hans`
- `繁中`、`繁体`、`TC`、`CHT`、`zh-Hant`、`zh-TW`、`zh_tw` → `zh-Hant`
- `日文`、`日本語`、`ja`、`jpn`、`ja-jp` → `ja`
- `英文`、`English`、`en`、`eng` → `en`
- 无法识别 → `und`

## 7. 匹配状态

表格中可能出现以下状态：

```text
已匹配
待修正
冲突
未处理
缺失视频
缺失字幕
```

匹配策略：

- 先按集数分组视频和字幕
- 同一集内按语言偏好排序
- 默认语言偏好：`zh-Hans`、`zh-Hant`、`ja`、`en`
- 同语言字幕优先格式：`ass`、`ssa`、`srt`、`vtt`
- 同一语言出现多个候选字幕时会标记为冲突

## 8. 整理规则

整理模式只有两种：

```text
复制整理
移动整理
```

复制整理：

- 原文件保留在原目录
- 视频和字幕复制到输出目录
- 不会删除原文件

移动整理：

- 视频和字幕一起移动
- 当前实现采用“先复制成功，再删除源文件”的方式
- 不会在复制失败时删除源文件

目标结构：

```text
<Anime Name> <Season>/
├─ videos/
│  └─ <Anime Name> S01E01.mkv
├─ subs/
│  ├─ zh-Hans/
│  │  └─ <Anime Name> S01E01.zh-Hans.ass
│  └─ ja/
│     └─ <Anime Name> S01E01.ja.srt
└─ anime-sub-map.json
```

文件名会按 Windows 规则清理：

- 替换 `< > : " / \ | ? *`
- 去掉末尾空格和点
- 避免 `CON`、`PRN`、`AUX`、`NUL`、`COM1`、`LPT1` 等保留名

## 9. 冲突处理

如果目标文件已经存在，应用不会静默覆盖。

整理计划会显示冲突项，用户可以选择：

- `跳过`：保留已有目标文件，不复制/移动当前文件
- `替换`：删除已有目标文件，再写入新文件
- `重命名`：生成类似 `文件名 (1).ext` 的新目标文件

默认冲突策略是 `跳过`。

## 10. MPV 播放

MPV 默认路径：

```text
mpv
```

这表示从系统 `PATH` 中查找 `mpv.exe`。

如果需要自定义路径，可以在右侧 `播放设置` 中修改，例如：

```text
D:\Tools\mpv\mpv.exe
```

启动 MPV 时使用参数数组，不拼接 shell 字符串。

概念上等价于：

```text
mpv <video-path> --sub-auto=no --sub-file=<primary-sub> --sub-file=<secondary-sub>
```

规则：

- 固定传入 `--sub-auto=no`
- 最多传入两个字幕文件
- 主字幕在前，副字幕在后
- 附加 MPV 参数以“每行一个参数”的形式填写

## 11. 项目保存

点击 `保存项目` 后，会在输出目录生成：

```text
anime-subtitle-manager.project.json
```

该文件保存：

- 项目名
- 季
- 视频目录列表
- 字幕目录列表
- 输出目录
- 主字幕语言
- 副字幕语言
- MPV 路径
- 附加 MPV 参数列表

## 13. 当前限制

当前版本仍有一些刻意保持简单的地方：

- 尚未实现项目配置文件的“打开/加载”入口
- 尚未解析真实视频分辨率、时长、文件大小、编码信息
- 表格中的 `编辑` 按钮目前主要由右侧详情选择器完成实际修改
- 右侧 `应用修改` 按钮暂未做额外提交逻辑，选择器修改会直接更新前端状态
- 浏览器预览不能执行本地文件系统和 MPV 操作
- 没有实现在线元数据、下载、字幕搜索或任何网络功能
- 没有实现 Linux/macOS 支持

  Jujutsu Kaisen S01E02.mkv

D:\AnimeTest\subs\
  [DBD-Raws][咒术回战][01][1080P][BDRip][HEVC-10bit][FLAC].SC.ass
  Jujutsu Kaisen S01E01.ja.srt
  Jujutsu Kaisen S01E02.zh-Hans.ass
  Jujutsu Kaisen S01E02.ja.srt
